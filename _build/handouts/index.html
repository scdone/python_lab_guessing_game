Guessing Game

Introduction

This exercise will show you how to write a command line application that allows users to play a “guess my number” game.

Setup

First, download the materials for this exercise from Frodo.

The folder you downloaded contains one file — game.py. Let’s get the lay of the land and see what it does.

One way to get a feel for what a Python file does is to run it!

$ python game.py

…nothing happened! What gives?

A convenient, quick way to check out a file is to output its contents to the terminal. That way, you don’t have to open another application just to skim through the code in a file (also, it’s good practice):

$ cat game.py
"""A number-guessing game."""

# Put your code here

Now it makes sense that nothing happened when we tried to run game.py. The file contains nothing but comments!

About cat

cat is short for concatenate. It concatenates the data in a file to the terminal.

Lastly, open game.py in your text editor.

Time To Build

In your text editor, add a print statement to game.py. You can print whatever you want (you’ll remove this line of code later). Remember to save your file. Your code should look something like this:

game.py
"""A number-guessing game."""

print('hi')

Now run your file!

$ python game.py

That’s your basic development workflow! You’ll always:

  • Open the files you want to edit in a text editor

  • Add code, save your changes

  • Run your file to make sure it works

Create a Guessing Game

About the Guessing Game

For the rest of this exercise, you’ll write a program in the game.py file that plays the classic number guessing game. The game will work like this:

  • The computer will choose a random number between 1–100 and ask the user to guess the number

  • Once the user guesses, the computer will tell the user if their guess was too high or too low

  • The game is over once the user guesses the computer’s number. When the game is over, the computer the total number of guesses the user made before getting the right answer.

Here’s the terminal output for an example game:

$ python3 game.py
Howdy, what's your name?
(type in your name) Jessica
Jessica, I'm thinking of a number between 1 and 100.
Try to guess my number.
Your guess? 50
Your guess is too low, try again.
Your guess? 80
Your guess is too high, try again.
Your guess? 60
Your guess is too low, try again.
Your guess? 70
Your guess is too high, try again.
Your guess? 63
Your guess is too low, try again.
Your guess? 64
Your guess is too low, try again.
Your guess? 67
Your guess is too low, try again.
Your guess? 69
Your guess is too high, try again.
Your guess? 68
Well done, Jessica! You found my number in 9 tries!

Here’s a rough pseudocode outline of the program that gave this output:

greet player
get player name
choose random number between 1 and 100
repeat forever:
    get guess
    if guess is incorrect:
        give hint
        increase number of guesses
    else:
        congratulate player

Gitting into Git

Git is an open-source software used for version control. Other version control technologies you may be familiar with are the Track Changes feature in Microsoft Word or See version history in Google Docs. Version control software takes snapshots of a document or documents as it changes over time. Then, it builds a timeline and allows you to jump to different snapshots — different versions — of the document(s).

Unlike with Google Docs and Microsoft Word where you can only track changes to one document at a time, Git allows you to track changes that have occurred in all files and directories that belong to one project. It’s not exclusive to tracking changes in code — it works with all file formats including images.

The other major difference between version control software for developers vs. software for non-developers is that — while Google Docs will create snapshots automatically for you — you’re in control of taking snapshots. In Git, snapshots are called commits.

Every time you commit your code, you’re preserving a snapshot in time. If you make a change you don’t like or mess something up, you can roll back to a previous commit. For those of you who play adventure games, the motto “Save Early, Save Often” should come to mind.

The following instructions will show you how to turn a regular project directory into a Git repository. In Git, a repository is a collection of files and directories. In other words, it’s a project directory where you’ve activated Git.

Initialize Git

Before you can start using Git, you’ll need to turn your boring, normal project directory into a Git repository! Make sure you’re in the folder guessing-game/, then run the command below:

$ git init

This will “activate” Git in your working directory! It sounds wild, but all that’s happening here is that Git created a hidden directory called .git in your working directory. You can see it for yourself by outputting all files — including hidden ones — in the terminal with ls:

$ ls -a

Create an Initial Commit

Now you’re ready to start using Git. Whenever you start a lab, you should make an initial commit before you write any of your own code. That way you’ll have a snapshot of what your exercise materials looked like when you recieved them in case you ever need to go back to the beginning.

Before you make your first commit, check the status of your repository:

$ git status
On branch main

No commits yet

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        game.py

nothing added to commit but untracked files present (use "git add" to track)

The message above tells us that Git isn’t tracking changes in game.py, which means we won’t be able to record any snapshots of it. Tell Git to track game.py with git add:

$ git add game.py

Run the git status command again. The status message should change:

$ git status
On branch main

No commits yet

Changes to be committed:
  (use "git rm --cached <file>..." to unstage)
        new file:   game.py

game.py is now staged which means it’s ready and waiting for you to make a commit. Finally! You can make that initial commit by running:

$ git commit -a -m "Initial commit"
[main (root-commit) 0734f92] Initial commmit
 1 file changed, 3 insertions(+)
 create mode 100644 game.py

Before making further changes, use the git status command again. Git should tell you that your working tree is clean — this means there are no other changes or files to commit.

$ git status
On branch main
nothing to commit, working tree clean

Sometimes, you’ll want to check the history of your commits to see what changes you’ve made. Try checking Git’s logs now. Press the q key when you’re finished reading your logs:

$ git log
(your previous commit messages)

Write descriptive commit messages, and this log will be an invaluable tool when you need to roll back to a previous state.

Let’s do this workflow one more time — except, this time, you’ll make a commit for code that you’ll have written and added to game.py. In the next section. you’ll start by adding code to greet the player.

Greet the Player

The first feature to implement is the greeting:

  • In VS Code, add code to game.py that displays a greeting to the player

  • Since you’ve added new code, run it to test that it works

  • If everything works and there are no errors, it’s time to commit your code!

Commit a New Feature

Before you do anything Git-related, you should always check the status of your repository:

$ git status
On branch main
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git restore <file>..." to discard changes in working directory)
        modified:   game.py

no changes added to commit (use "git add" and/or "git commit -a")

The message above tells us that Git is tracking game.py and it has noticed that game.py has changed.

You’re ready to make another commit! You can make commits as often as you’d like, but you should always commit after you’ve added more code that works. If you make this a habit, you’ll always have snapshots of working code to rollback to. You should also write a brief but informative commit message to describe the changes you’ve just made:

$ git commit -a -m "Display player greeting"

Get the Player’s Name

After “greet player,” our pseudocode outline says “get player name”.

  • Write code in game.py that get the player’s name and save it to a variable

  • Run your code to test it

  • Once everything works, make another commit. Remember to check Git’s status before committing.

As you get more comfortable with Git, you won’t need to do a git status at every step. We’re only having you do that here so you can see what’s happening at each step and get used to the messages that Git returns. But do commit early, commit often, and when in doubt, use git status and git log to confirm that reality is what you think it is.

Repeat this process for the remaining steps in the pseudocode outline, testing your code at each step. Whenever you have the code for a step working, commit it. The next thing you’ll do is choose a random number.

Choose a Random Number

Python comes with a random module that contains functions for generating random numbers. To access the code in a module, all you have to do is import it. You can put import statements anywhere, but it’s a good practice to put it at the top of your file like this:

game.py
"""A number-guessing game."""

import random

# (...snippet)

To import any module, all you have to do is write import followed by the name of the module. After that, you can use any function included in the module like so:

random.function_name()

Alternatively, if you know the name of the exact function you want to use, you can import that specific function like so:

game.py
"""A number-guessing game."""

from random import function_name

# (...snippet)

Using the syntax above, you’ll be able to omit the module name when you call function_name:

function_name()

Now that you know how to import functions from modules, check out the official docs on the random module here: The Python Standard Library: random. Find a function that’ll help you generate a random number between 1–100.

Turning stuff into numbers

If you check out the chapter of the Python docs on numeric types that we linked above, you’ll see a table of operations after the 4th paragraph. Here’s an excerpt from that table:

Operation

Result

int(x)

x converted to an integer

So if we want to convert something to an integer, all we have to do is call the int function. For example:

>>> num = '100'  # this is not a number, it's a string
>>> num = int(num)
>>> num
100
