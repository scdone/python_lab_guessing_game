Rubric

topics

Git basics and python basics including data types, formatting strings, type conversion, while loops, conditionals, and user input

further study concepts

type checking, counters (in loops)

requires

Beginner-level Python, Terminal

time

1-3 hours (intro material takes about 1.5~ hours)

authors

Christian Fernandez

Goals

Students will create a number guessing game that can be played from the command line. This exercise demonstrates input validation, basic Python control structures, using the Terminal to run Python scripts, and basic Git commands.

Length

This is an all day exercise. Fastest pair in Summer 2015 finished in 2 hours (including intro material). Several other finsihed around that time. The slowest pair was about 2.5 hours.

Gotchas

Students may be tempted to skip the intro work because they think it’s an optional part of the exercise.

raw_input does not produce an int. If every guess is “too high”, that’s probably the problem.

Some people are making directories called “subl” when given this instruction “Create your project file. ‘subl guessing.py’ will probably get you started. ;)” Perhaps we should clarify that.

Potential for an endless while loop if they don’t break. Hopefully students will remember to Ctrl-C in that case, but if they do Ctrl-Z, remember to use fg to pull up their process.

Git is scary, I have few suggestions for this. Try to act like it’s no big deal while maintaining caution around using git reset –hard

Further Study: try and except for converting to int is tricky. Have students google “Errors and Exceptions in Python” to get the docs.python.org page. Have them read it.

Code Reviews

Code Review 1

Students have finished creating guessing game that:

  • Asks for user’s name

  • Greets user and prompts them to enter a number

  • Cycles through until the user guesses the right number (giving hints about whether the guess is too high or too low)

  • ends when the user guesses the right number

They have learned about the purpose of a “main loop.” They’ve also been using git and were asked to do a git reset –hard.

Asked to use/look into the Python int function and random module

Check in about pair programming – who was last driving, who was navigating?

Other things to check/ask about:

  • randint(1, 100) – includes the 2nd number given, as opposed to functions like range() that go up to but not including the second argument.

  • different types of string formatting (.format(), %s, string concatenation)

Further Study, Code Review 1

If the user inputs a number that isn’t between 1-100 as requested, mock them for their crimes and ask them to enter a valid number.

Further Study, Code Review 2

If the user inputs something that is not a number, mock them for their crimes and ask them to enter a valid number. Hint: a good way to do this is to learn about Python’s try and except statements.

  • Did students put a specific error in their except clause? (ValueError)? How did they figure out it was a ValueError?

  • Ask about how try and except work. What did they wrap in their try statement and whatw as in the except statement?

Further Study, Code Review 3

Ask the user if they would like to play again and restart the game rather than exiting.

  • Break and continue statements
